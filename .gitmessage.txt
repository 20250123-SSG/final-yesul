### -- 한 줄 띄우는 용도 아래줄은 작성 또는 지우지 마세요. (본문/꼬릿말 구분) --

### 꼬릿말 : <꼬리말 키워드> + #이슈번호  (ex. Refs: #5)

# ------------------------------------------------
#                 < 타 입 >
#         feat : 새로운 기능 추가
#         fix : 버그 수정
#         refactor : 리팩토링 (기능변경없이 코드 개선)
#         docs : 문서 변경
#         test : 테스트 코드 변경
#         build : 빌드 파일 수정
#         perf : 성능 개선
#         style : 코드 스타일, 포맷, 정렬, 공백 등을 수정
# ------------------------------------------------
#                <꼬리말 키워드>
#         Refs:       -> 이슈랑 연결만 원하는 경우
#         closes      -> 완료 후, 이슈닫기
#         fixes       -> 버그 고친 후, 이슈닫기
# ------------------------------------------------
